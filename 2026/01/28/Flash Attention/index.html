# 1. Flash Attention

Flash Attention（FA）是训练大模型默认采用的技术。原论文《FlashAttention: Fast and Memory-Efficient Exact Attention with IO-Awareness》由斯坦福大学于 2022 年提出，论文标题的 “IO-Awareness” 指的是 Flash Attention 改进 I/O 操作以减少内存访问延迟，从而提升整体计算速度。

Flash Attention 保证 exact attention 计算，也就是说它的计算结果和标准的 attention 计算结果完全一致。

# 1.1. 标准 self-attention

在标准的 self-attention 计算中，给定查询（Q）、键（K）和值（V）矩阵，计算过程如下：

Attention(Q,K,V)=softmax(QKTdmodel)V\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_{model}}}\right)V Attention(Q,K,V)=softmax(dmodel​​QKT​)V

其中，dmodeld_{model}dmodel​ 是模型编码向量的维度

# 1.1.1. 标准 self-attention 在硬件的运算

在硬件上，标准的 self-attention 计算通常涉及以下步骤：

  1. 计算 QKTQK^TQKT
    • 加载：从高带宽存储（HBM）中分块（by blocks）读取矩阵QQQ 和 KKK 到片上内存（SRAM）。
    • 计算：在片上内存中计算 QKTQK^TQKT。
    • 存储：将中间结果写回 HBM 中。
  2. 计算 softmax
    • 加载：从 HBM 中读取 QKTQK^TQKT 到片上内存。
    • 计算：在片上内存中计算 softmax。
    • 存储：将 softmax 结果写回 HBM 中。
  3. 计算最终结果
    • 加载：从 HBM 中读取 softmax 结果和矩阵 VVV 到片上内存。
    • 计算：在片上内存中计算最终的注意力输出。
    • 存储：将最终结果写回 HBM 中。

# 1.1.2. roofline model

roofline model 是一种用于分析计算性能的模型，它将计算性能与内存带宽进行对比，以确定系统的瓶颈。

上图展示了 roofline model 的主要思想。

  • PPP 计算性能（FLOPS）表示系统每秒可以执行的浮点运算次数，
  • III 计算强度（FLOPS/Byte）表示每读取字节数据可以执行的浮点运算次数。

I=π/βI=\pi / \betaI=π/β, 其中 π\piπ 是每秒浮点运算次数，β\betaβ 是每秒读取内存大小。如果是硬件则可计算最大计算效率，如果是 model 则可计算实际计算效率。

Roofline Model 中算力决定 “屋顶” 的高度（图中绿色线段），带宽决定 “房檐” 的斜率（图中红色线段）。Roofline Model 划分出的两个瓶颈区域为：

P={β⋅I,when I<ImaxMemory Boundπ,when I⩾ImaxCompute BoundP = \begin{cases} \beta \cdot I, & \text{when } I < I_{max} & \color{red}{\text{Memory Bound}} \\ \pi, & \text{when } I \geqslant I_{max} & \color{green}{\text{Compute Bound}} \end{cases} P={β⋅I,π,​when I<Imax​when I⩾Imax​​Memory BoundCompute Bound​

  • 计算约束 (Compute Bound) —— 此时 HBM 访问所花费的时间相对较低，不管模型的计算强度 III 多大，它的理论性能 PPP 最大只能等于计算平台的算力 π\piπ。例如，具有较大维度的矩阵乘法和具有大量通道的卷积。
  • 带宽约束 (Memory Bound) —— 此时模型位于 “房檐” 区间，模型理论性能 PPP 大小完全由计算平台的带宽上限 β\betaβ（房檐的斜率）以及模型自身的计算强度 III 所决定。例如，elementwise 操作（如 activation、dropout 等）和规约操作（如 sum、softmax、batch normalization、layer normalization 等）。

在 self-attention 中，它的计算瓶颈主要是 Memory Bound。

# 1.2. Flash Attention 的优化

# 1.2.1 Q, K, V 分块计算

Flash Attention 通过将 Q、K、V 矩阵分块计算，减少了中间结果的存储需求，从而降低了内存访问的开销。

这里先跳过 Softmax 操作下一小节有详细说明

  1. 首先我们从 HBM 中分块读取 QQQ 的前两行，KTK^TKT 的前三列。然后将读取的块传入 SRAM 中进行矩阵乘法 QKTQK^TQKT 的计算得到 SSS 。得到的中间结果 SSS 并不存入 HBM 中，而是直接在 SRAM 里与 VVV 的分块进行下一步计算，最终得到 OOO 的前两行。但是由于 KKK 和 VVV 的分块是独立的，因此需要多次读取 KKK 和 VVV 的分块来完成最终结果的计算。先用浅色表示这不是最终结果只是缓存结果。
  1. 接着 KKK 和 VVV 的分块还保留在 SRAM 中不变，从 HBM 中读取 QQQ 的中间两行，然后经过同样的计算，得到 OOO 的第三行和第四行的中间结果
  1. 然后经过同样的计算，得到 OOO 的最后两行的中间结果

  1. 此时 QQQ 的所有行都已经处理完毕，切换 KKK 和 VVV 的分块，从 HBM 中读取 KTK^TKT 的后续三列和 VVV 的后续三列进行计算。而最开始的前两行 OOO 的中间结果还保留在 SRAM 中，因此这个时候可以合并这次计算的 OOO ，得到前两行的最终结果。

  2. 重复上述过程，直到所有 KKK 和 VVV 的分块都处理完毕，最终得到完整的 OOO 矩阵。

可以看出，通过分块计算，大大减少了内存访问的开销，提高了计算效率。

# 1.2.2.Softmax 分块

原始 softmax 的计算公式如下：

Softmax(xi)=exi∑k=1Nexi=p(x)ℓ(x)\text{Softmax}(x_i) = \frac{e^{x_i}}{\sum_{k=1}^{N} e^{x_i}}=\frac{p(x)}{\ell(x)} Softmax(xi​)=∑k=1N​exi​exi​​=ℓ(x)p(x)​

但是在实际计算中，直接计算 exie^{x_i}exi​ 可能会导致数值不稳定的问题，尤其是当 xix_ixi​ 的值较大时。因为，在半精度浮点数（FP16）表示中，FP16 所能表示的最大数值是 65536，这意味着当 x≥11x ≥ 11x≥11 时，exie^{x_i}exi​ 就超过了 FP16 所能表示的最大范围了

因此，通常会对 softmax 进行数值稳定化处理，具体做法是减去输入的最大值：

Softmax(xi)=exi−max⁡(x)∑k=1Nexk−max⁡(x)\text{Softmax}(x_i) = \frac{e^{x_i - \max(x)}}{\sum_{k=1}^{N} e^{x_k - \max(x)}} Softmax(xi​)=∑k=1N​exk​−max(x)exi​−max(x)​

其中

  • m(x)=max⁡(x)m(x) = \max(x)m(x)=max(x)
  • x(1)x^{(1)}x(1) 和 x(2)x^{(2)}x(2) 分别表示分块后的两个子矩阵

# 1.3. Flash Attention2

  1. 增加了一些工程上的优化，比如减少了非矩阵乘法的计算，将 QQQ 改为外循环，KKK , VVV 改为内循环，更进一步减少了对 HBM 的读写。
  2. 判断一个分块是原始矩阵的上三角部分，如果是则不需要进行 attention 的计算，从而更进一步减少了计算量。

# 1.4. Flash Attention3

  1. 利用硬件异步性（Asynchrony）FlashAttention-3 重新设计了算法，利用 Hopper 架构特有的 TMA (Tensor Memory Accelerator) 和 WGMMA (Warpgroup Level Matrix Multiply-Accumulate) 指令来消除计算瓶颈。
  2. FP8 低精度优化 FlashAttention-3 引入了对 FP8（8 位浮点数） 的原生支持，不仅大幅提升了计算速度（最高达 1.2 PFLOPS），还解决了低精度带来的精度损失问题。
  3. 其他工程优化持久化调度（Persistent Scheduler）：在处理当前算块的输出时，提前开始加载下一个子问题的 Q,K,VQ, K, VQ,K,V 数据，优化了短序列情况下的性能表现。变量长度序列支持：针对 Llama 3 等模型中常见的变长序列训练进行了专门的内核优化。精度对齐：在 FP16 模式下，通过将中间结果（如 Softmax 缩放）保持在 FP32 中，确保了与 FlashAttention-2 相同的数值准确度，甚至优于标准 Attention 实现。
